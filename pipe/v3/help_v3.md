# 📌 Pipeline Guide — v1, v2, and v3

This document covers all three versions of the AI reasoning report generator
and explains what each version does, what problems it solves, and how to run it.

## 📖 What Each Version Does

### V1 — `vision_reasoning_report.py`
**One image → one prompt → one response.**

The original implementation. For each misclassified image, a single VLM prompt
is sent containing the image, all metadata (true label, predicted label,
distortion type, confidence), and a request for 7 structured sections. Each
image is analysed in complete isolation. The "blind spot summary" is assembled
by joining root causes with a pipe character — not synthesised by the model.

Best for: quick single-image inspection, minimal server load.

---

### V2 — `vision_reasoning_report_v2.py`
**All images for a distortion type → two-turn reasoning → synthesised analysis.**

Two key upgrades:
1. **Multi-image batching**: all 3 sampled images for a distortion type are sent
   in one VLM call. The model can compare typical vs outlier failures.
2. **Two-turn recursive structure**: Turn 1 withholds labels (forcing genuine
   visual grounding). Turn 2 reveals labels and asks for cross-image synthesis.

Section 5 (blind spot summary) is now generated by the model comparing images,
not by string concatenation.

Best for: standard analysis runs where the vLLM server is already warm.

---

### V3 — `vision_reasoning_report_v3.py`
**Full agentic pipeline with cross-run memory and data-verified reasoning.**

Five new capabilities:

**① Semantic deduplication** (`v3/image_sampler.py`)
Before sending images to the VLM, computes pairwise cosine similarity between
pixel embeddings. Removes near-duplicates (threshold: 0.96 cosine similarity),
ensuring the batch always contains maximally diverse failure examples.

**② RAG — cross-run failure memory** (`v3/rag.py`)
Builds a vector index of every misclassified image across ALL training runs,
stored as `reports/rag_index.npz`. At query time, retrieves the K most
visually similar past failures and injects them into Turn 2 as historical
precedent. The model can reason: "this blur/cat failure matches 8 prior cases
with the same root cause — or — this is a new pattern not seen before."

**③ LangGraph agentic loop** (`v3/graph.py`)
4-node state machine: `observe → hypothesise → verify → conclude`.
The `verify` node calls 3–4 stat tools (confusion count, epoch trend, top
confusion pairs for the distortion type) to check whether the hypothesis is
consistent with the data. If not, the graph loops back to `hypothesise` with
the tool results as corrective context (max 2 iterations).

**④ Pydantic structured output** (`v3/schemas.py`)
Turn 2 uses `PydanticOutputParser(Turn2Analysis)` to force JSON-schema
compliant responses. Field-level validation. Failures are explicit (the
raw text is shown) rather than silently producing `_raw`.

**⑤ Dynamic recommendations** (`v3/renderer.py`)
Section 6 calls the LLM with the actual failure distribution + root causes +
confusion pairs from this run. The model generates 5 prioritised, data-specific
recommendations — no hardcoded strings.

## ▶️ Running V3

### Mode A — Full run (VLM + RAG + agent)

```bash
# Terminal 1 — vLLM server
cd ~/vision_dev_project && source venv_vision/bin/activate
VLLM_USE_V1=0 HUGGINGFACE_HUB_CACHE="/mnt/data/pratik_models" \
python3 -m vllm.entrypoints.openai.api_server \
  --model Qwen/Qwen2.5-VL-7B-Instruct \
  --quantization bitsandbytes \
  --gpu-memory-utilization 0.4 \
  --max-model-len 2048 \
  --enforce-eager \
  --port 8000
# wait for: INFO:     Application startup complete.

# Terminal 2 — v3 report
cd ~/vision_dev_project/pipe && source ../venv_vision/bin/activate
python3 vision_reasoning_report_v3.py
# output: ../ai_reasoning_summary_v3.md
```

### Mode B — Stats + RAG only (no GPU / no VLM server)

```bash
python3 vision_reasoning_report_v3.py --no-vlm
```
Builds the RAG index, runs semantic deduplication, and writes a stats-only
report with data-aware fallback recommendations.

### Mode C — Full run without RAG

```bash
python3 vision_reasoning_report_v3.py --no-rag
```
Skips the RAG index entirely. Faster cold start; no cross-run context.

## 🎛️ All Flags

```
# Inherited from v1/v2
--logs-dir PATH     Logs folder (default: pipe/logs)
--report PATH       distortion_report.json path
--output PATH       Output .md path (default: ai_reasoning_summary_v3.md)
--samples N         Images per distortion type (default: 3)
--training-log PATH Override: specific training_log JSON
--misclassified PATH Override: specific misclassified JSON
--port PORT         vLLM server port (default: 8000)
--model NAME        vLLM model name
--seed N            Random seed (default: 42)
--no-vlm            Skip all VLM calls

# New in v3
--no-rag            Skip RAG index build/load
--rebuild-rag       Force rebuild RAG index even if cached .npz exists
--rag-index PATH    Override path to rag_index.npz
--no-tool-trace     Hide tool verification results from report
```

## 📋 V3 Report Structure

```
1. Training Summary
2. Failure Distribution by Distortion Type
3. Top Confusion Pairs
4. Per-Distortion Agent Analysis  (one subsection per type)
   ├── Stats table
   ├── Images analysed table (post-dedup)
   ├── 🗃️ RAG context (retrieved similar past failures)
   ├── Turn 1 — visual observations block
   ├── 🔧 Verification tool results (epoch trend, confusion counts)
   └── Turn 2 — structured analysis (Pydantic-parsed)
       🔗 Shared Failure Pattern
       ⭕↔⚠️ Typical vs Outlier
       ❌ What Misled the Model
       📊 Confidence Assessment
       🎯 Root Cause
       🗃️ RAG — Novel vs Known Pattern
5. Model Blind Spot Summary (agent-synthesised, tool-verified)
6. Actionable Recommendations (LLM-generated from live data)
```

## 🔁 Full Pipeline Flow (all steps)

```bash
# Step 1 — train
python3 test.py
# → logs/training_log_*.json  +  logs/misclassified_*.json

# Step 2 — cluster
python3 distortion_diagnostic_report.py \
  --base-dir logs/misclassified_YYYYMMDD_HHMMSS.json \
  --output   reports/distortion_report.json
# → reports/distortion_report.json  +  reports/distortion_clusters.png

# Step 3 — v3 report (with vLLM server running in Terminal 1)
python3 vision_reasoning_report_v3.py
# → ai_reasoning_summary_v3.md  +  reports/rag_index.npz (first run)
```

On the first run, V3 builds `rag_index.npz` from all available misclassified
JSONs. Subsequent runs load from cache unless a newer JSON is detected
(automatic staleness check).

## 🧪 Troubleshooting

**RAG index takes a long time to build**
Embedding extraction processes every base64 image in every misclassified JSON.
With thousands of failures across multiple runs this can take 1–2 minutes.
The index is cached — subsequent runs load in <1 second.

**Pydantic parse fails for a distortion type**
The raw VLM text is stored and displayed verbatim. Check `--max-model-len` on
the vLLM server — if 2048 is too short for the combined Turn 2 prompt
(RAG context + tool results + images), raise it to 4096.

**Agent loops twice then concludes without structured output**
The `verify → hypothesise` revision runs at most `AGENT_MAX_ITERATIONS=2` times
(set in `v3/config.py`). After that, `conclude` runs with whatever
`hypothesis_text` was produced. Raise the limit in config if needed.

**`--no-rag` is much faster**
The RAG index build decodes every base64 image in every training run. If you
only care about the current run, `--no-rag` skips this step entirely.

**Dynamic recommendations look generic**
If the VLM call for recommendations fails, the data-aware fallback in
`v3/renderer.py` generates recommendations ranked by actual failure count.
These are more relevant than the old hardcoded table but less detailed than
the full LLM-generated version.
